SCypher

XOR-based BIP39 Seed Cipher

One master. Many derived wallets. Encrypted. Reversible. Secure.

What is SCypher?

SCypher is a tool that enables deterministic encryption of BIP39 seed phrases with a password, generating new valid BIP39 phrases without ever exposing the original master seed.

🔐
One master = many secure wallets
🔄
Reversible with same password
🧩
Unique BIP39 output per user
🛡️
Offline use, no server required
🚫
One leak doesn't compromise the rest
🧠
No need to store derived seeds – can regenerate on demand

How SCypher Works

Seed + Password + Iterations → XOR → Encrypted Seed

Each BIP39 word represents 11 bits of entropy
2
Generate keystream
From password + iterations using SHAKE-256
3
XOR operation
Reversible encryption between keystream and seed
4
Convert back to BIP39
Valid seed phrase with correct checksum

Personal Use Cases

🔒 Secure Storage
Scenario A:
  • Encrypt your seed with a password
  • Store the encrypted seed physically
  • Use password to recover original when needed
  • Master seed never leaves secure storage
🌳 Multi-Derivation
Scenario B:
  • Keep ONE master seed physically secure
  • Derive multiple seeds with different passwords
  • One seed → many wallets (exchange, DeFi, savings)
  • Rotate wallets by changing passwords

Enterprise Use Cases

🏢 Organizational Delegation & Access Control
  • Organization holds one master seed in secure cold storage
  • Uses internal passwords per user/device/department (never shared externally)
  • Generates valid BIP39 phrase per user without exposing master
  • Each employee receives only their derived phrase
  • If a user wallet is compromised, others remain completely safe
  • Perfect for delegation: signing authority, fund management, smart contracts
  • Easy access revocation when employees leave
  • No need to store hundreds of derived seeds - regenerate on demand

BIP39 Passphrase vs SCypher

Practical Use Case BIP39 Passphrase SCypher Encryption
Requires the original (master) seed ✅ Yes, always ❌ No, each derived seed is self-contained
Generates different usable wallets ✅ Yes ✅ Yes
Output format 🔐 Same seed + hidden passphrase logic 🧩 New valid BIP39 seed (independent)
Applying BIP39 passphrase 🔁 Already built into derivation, not reapplicable ✅ Can optionally apply passphrase to the derived seed
Regeneration / Exposure Risk 🔁 Requires master seed + passphrase. If master leaks, brute-force is possible ✅ Can regenerate or rotate safely. One leak doesn't expose master or others
Risk when user leaves organization 🔴 If master seed was leaked, all wallets are at risk of brute-force 🟢 Only their derived seed is exposed, others and master remain safe

Security & Attack Resistance

🛡️ Design Security
  • XOR reversible encryption with password-derived keystream
  • SHAKE-256 (current) and Argon2id (future) as KDFs
  • Output always valid BIP39 (entropy + checksum)
  • Derived seeds don't reveal master seed
🔍 Attack Resistance
  • Even if attacker guesses correct password, cannot identify the master
  • Each derived seed appears as valid standalone seed
  • No verification mechanism - attacker can't confirm correct guess
  • Argon2id adds powerful brute-force resistance

Security by design: When passwords are strong and diverse, reverse-engineering becomes computationally unfeasible, even with multiple compromised derived seeds.
